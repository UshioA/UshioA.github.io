缓冲区炸弹

发表于 2021-11-07 更新于 2021-11-24 分类于 爬行史

本篇记录缓冲区溢出炸弹实验的求解过程.

大体感想

这次的缓冲区溢出实验, 相比之前的二进制炸弹拆解显得容易一些, 可能是直接攻击思维上比较好理解吧, 毕竟上次还得真的用gdb不断地反汇编来了解炸弹的真实意图, 而这次直球攻击就行了. 不过, 想要攻击得手还是得先了解一下堆栈结构.
先抄书.

在C语言程序执行过程中, 当前正在执行的过程(即函数)在栈中回形成本过程的栈帧. 一个过程的栈帧中除了保存EBP和被调用者保存寄存器的值外, 还会保存本过程的非静态局部变量和过程调用的返回地址. 如果在非静态局部变量中定义了数组变量, 那么, 有可能在对数组元素访问时发生超越数组存储区的越界访问. 通常把这种数组存储区看成是一个缓冲区, 这种超越数组存储区范围的访问称为缓冲区溢出.

简单来说, 因为代码和数据混合放置, 在内存中开辟的缓冲区事实上会和代码挨在一起. 如果没有溢出, 当然不会有任何危险, 但是如果不加确认的直接写入缓冲区而且还发生了溢出, 就会覆盖原有的代码, 数据, 以致程序发生不可预料的行为. 如果恶意攻击者利用这一点, 就可以达到攻击的目的.

另一个重要的常识

在i386机器中, 栈从高地址向低地址增长. 虽然这个是常识, 但是我总是记不住.

那么, 接下来就记录一下解题的过程吧.

Smoke

Smoke是本次实验的第一个阶段. 它要求你利用缓冲区溢出使得程序跳转到Smoke. 这个比较容易. 主函数会调用test过程, 调用之前, 会将返回地址压入栈中, 接下来, 又将ebp压入栈中, 然后将esp的值赋给ebp. 梳理一下这个过程. 首先将返回地址压入栈中, 接着又将ebp压入栈中,稍作思考可以发现 ,此时, 返回地址在栈里面就处于&ebp + 4的位置, 这是因为栈从高地址向低地址增长, 而ebp入栈较晚. 接下来一步更是重量级, 它等价于ebp = &ebp, 这样, 栈里面ebp原来的值的地址就得到了, 我们就可以通过它来确定返回值的位置了. 那么, 如果知道缓冲区的大小, 我们就可以通过让缓冲区溢出的方式覆盖返回值, 使得程序想跳到哪, 就跳到哪.

考虑如何达到这个目的. 首先考虑从什么地方开始攻击这个程序. 唯一能够写入的地方是缓冲区, 因此只能从这个方面入手. 进一步考虑, 向缓冲区内写入恶意代码之后, 该如何执行这些代码呢? 因为这台机器是顺序地执行代码的, 必须想一个办法让它跳转到缓冲区开头执行才好. 为了达到这个目的, 可以覆盖返回值地址, 让它返回到缓冲区开头来执行我们写入的恶意代码. 这几乎是唯一的选择, 因为写入这些恶意代码并不能马上见效, 我们的缓冲区在机器中可能和实际执行的代码相距甚远, 机器仍然会按照它自己的步调继续往下执行直到遇到ret指令, 发生一个跳转, 而这正是我们有机可乘的地方.

移除最多的同行或同列石头

发表于 2021-11-05 更新于 2022-03-05 分类于 挣扎题解

母题是移除最多的同行或同列石头.

题目要求为, 如果石头有和自己同行或者同列的, 就可以把它给移走. 考虑如何将这个描述转化为可以写成代码的形式.

先自己来做这件事情. 为了做到它, 可以想象一个二维的坐标系, 把石头画在坐标系的点上. 为了确定怎么移走石头, 一个有效的做法是把相邻的石头用一根线连起来, 然后把这根线上面除了端点的点全删掉, 统计删去的点数. 为了把所有能删的点都删掉, 上述删除过程可能进行数次.
简化这个情形, 可以将石头看成散列的点, 如果满足某个性质就把它们连起来, 这就会形成几个连通分支, 则题目的意思就是把连通分支删成孤立的点. 进一步的, 只要知道连通分支的个数, 则可以用点的总数减去连通分支的数量, 就能得到结果.

上课刚讲并查集老师让我们课下自己多了解你就出并查集的题目真有你的显然, 并查集可以方便的做到这一点. 因此, 首先写一个并查集的轮子, 然后在程序开头初始化一个并查集. 接着接受用户的输入, 将这些点保存起来, 然后搜索, 当前一个点和后一个点是相邻的, 就将它们连接起来. 并查集中会自动更新当前连通分支的数目. 所有的点都做完连接之后, 就可以直接得到题目所要求的结果了.

最大子树和

发表于 2021-11-04 分类于 挣扎题解

首先贴一个题目要求:

一天小蓝鲸早晨骑车去上课，路上见到一个老伯正在摘果树上的果实，其中有的刚刚成熟，有的已经腐烂，顿时想到了一个有关修剪果树的问题，希望通过修剪果树得到一棵最好的果树。因此，他想到了如下的问题：

现有一棵果树，上面共连有N颗果实，共有N-1条枝干将果实连在一起，并且未修剪时每颗果实是通过枝干连在果树上的。每颗果实都有一个“成熟指数”，该数越大说明这颗果实越成熟；也有“成熟指数”为负数的，说明这颗果实已经腐烂。所谓“修剪”，意为：去掉其中的一条枝条，这样一棵果树就成了两棵，扔掉其中一棵“成熟指数”小的，保留“成熟指数”大的果树。经过一系列“修剪“之后，还剩下最后一棵果树（也可能是只有一个果实）。

注：“修剪”只能剪掉连接的枝干，不能只摘下果实。

因此，通过一系列“修剪”（也可以什么“修剪”都不进行），使得剩下的那棵果树上所有果实的“成熟指数”之和最大，我们称之为最好的果树。

此题的母体为洛谷P1122 最大子树和, 相比原题做了简化, 主要是调整了节点之间相邻关系的输入顺序从而可以直接从输入建立起一棵树. 利用这个可以对树只进行一次dfs就能找到所有的候选解.

尽管此题在洛谷上被称作入门题, 但是还是给予了本菜鸡极大的震撼, 做了一晚上才调出来.

从一个只知道无脑递归的人的视角, 只能想到怎么分解一个问题. 假如将目光对准一个挂着好多个苹果的枝条, 人是犹豫的, 这是因为他不能确定保留这一串苹果是赚了还是亏了. 但是如果只关心长在枝条末尾的苹果(称为树叶苹果), 则它的值就是它自己, 可以很轻松的判断保留它是赚还是亏.

这样, 我们就得到一个basecase, 即当该苹果是树叶苹果的时候, 可以直接返回苹果的成熟度作为最好的果树, 注意此时就算是烂了的苹果也是最好的, 因为对于一个孤儿, 它只有自己.

然后再考虑如何逐步筛选出最好的果树. 前面已经讨论了树叶苹果的最优解, 对于树叶的爸爸, 事情也很简单, 如果树叶成熟度大于0, 就把它保留, 反之丢掉. 这样, 每个树叶的爸爸都可以确定一个最优解.

由此上溯就能确定每个节点的最优解, 此处的最优解意思是只考虑保留该节点和该节点所有的子树所得到的最大子树和.

然后可以寻找所有这些和的最大值返回, 就是所需的最大子树和.

考虑到一次从根开始的dfs其实就可以生成所有的这些和, 采取一个数组保存这些结果, 这样能大大减小时间和空间复杂度.

最后贴一个垃圾代码. 写的非常的糟糕, 很多地方都有不必要的行为, 因为是修修补补搞出来的. 不过至少它能过.
